# ADR-007: Локальный in-memory стейт для избранного

**Статус:** `accepted`
**Дата:** 2026-04-09

## Контекст

Backend API для избранного не реализован (backend issue #22). Нужно дать пользователю рабочую кнопку «В избранное» прямо сейчас, не блокируя релиз фичи на бэкенд.

## Решение

- В `AppSession` добавлен приватный `_favorites: MutableSet<String>` (хранит eventId).
- Методы `isFavorite(eventId)` и `toggleFavorite(eventId, add)` доступны из любого Composable.
- `EventDetailScreen`: кнопка ♡/♥ в правом верхнем углу hero-изображения, анимированная смена цвета (серый → красный).
- Стейт `isFavorite` инициализируется из `AppSession.isFavorite(eventId)` — повторное открытие того же события сохраняет выбор в рамках сессии.

## Альтернативы

- **`DataStore` / `SharedPreferences`** — сохранение между сессиями, но требует платформенной реализации (expect/actual) и усложняет код без бизнес-необходимости пока бэкенд не готов.
- **Ждать бэкенда (#22)** — блокирует видимую фичу на неопределённый срок.

## Последствия

- Избранное **сбрасывается при перезапуске** приложения. Приемлемо как временное решение. ✓
- После реализации бэкенда (#22): заменить `AppSession._favorites` на вызовы API, сохраняя интерфейс `isFavorite`/`toggleFavorite`. Переход прозрачен для UI.
- Нет синхронизации между устройствами и нет `isFavorite` в ленте (поле требует бэкенда).
