# ADR-009: Автономный режим — in-memory кеш билетов

**Статус:** `accepted`
**Дата:** 2026-04-09

## Контекст

При отсутствии интернета приложение ранее показывало пустой экран билетов. Требование: показывать билеты даже без сети, чтобы пользователь мог предъявить QR-код на входе.

## Решение

- `AppSession.cachedTickets: List<TicketDto>` — обновляется при каждом успешном вызове `getMyTickets`. Живёт в памяти процесса.
- `AppSession.isOffline: Boolean` — `mutableStateOf`, устанавливается в `true` при любом исключении сети в `TicketsScreen` и `FeedViewModel`.
- При ошибке сети в `TicketsScreen`: показываем `cachedTickets` + `OfflineBanner`.
- `OfflineBanner` — переиспользуемый composable (жёлто-красная полоса с текстом), используется и в `FeedScreen`.
- Фильтр «только актуальные» (`usedAt == null`) уже применяется — архивные билеты в офлайн-режиме тоже доступны.

## Альтернативы

- **`DataStore` / `SharedPreferences` (постоянное хранилище)** — выживает после перезапуска, но требует `expect/actual` для KMP и добавления зависимости. Планируется как следующий шаг.
- **SQLite / Room** — избыточно для списка из ~10 билетов.
- **Ktor caching plugin** — кеширует HTTP-ответы, не даёт контроля над TTL и форматом данных.

## Последствия

- Билеты доступны офлайн **в рамках сессии** (пока процесс жив). После перезапуска кеш пуст. ✓ Приемлемо для MVP.
- `isOffline` реактивен — все экраны, читающие `AppSession.isOffline`, автоматически получают баннер. ✓
- **Известное ограничение:** после перезапуска без сети кеш пуст. Решается постоянным хранилищем (issue #63 частично адресует это).
